## 1. テスト方針

### テストレベル

| レベル | 範囲 | 担当 | 備考 |
|--------|------|------|------|
| 単体テスト（UT） | 各関数・メソッド単位 | 開発者 | 自動テスト（カバレッジ80%以上目標） |
| 結合テスト（IT） | API単位の入出力検証 | 開発者 | 本書のAPIテストケースが該当 |
| システムテスト（ST） | 画面操作を通じたE2E検証 | QA | 本書の画面テストケースが該当 |
| 受入テスト（UAT） | 業務シナリオベースの検証 | 業務担当者 | 本書のシナリオテストが該当 |

### テスト観点

| 観点 | 説明 |
|------|------|
| 正常系 | 想定通りの入力で期待結果が得られること |
| 異常系 | 不正入力・エラー条件でバリデーションが機能すること |
| 境界値 | 文字数上限・日付範囲の境界で正しく動作すること |
| 権限 | ロール別に適切なアクセス制御がされること |
| 状態遷移 | データの状態変化が正しいこと（作成→更新→削除） |

### 合否判定基準

- 全テストケースの実施率: 100%
- 重大度「高」のバグ: 0件
- 重大度「中」のバグ: 未修正3件以下
- 重大度「低」のバグ: 未修正10件以下
